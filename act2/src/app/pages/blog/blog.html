1. Justificación del framework

¿Para qué sirve Angular?

Angular es un framework de desarrollo frontend creado por Google. Sirve para construir aplicaciones web modernas, interactivas y escalables utilizando TypeScript. Angular destaca por ofrecer una arquitectura robusta que favorece la creación de interfaces modulares y mantenibles.

¿Qué aplicaciones tiene?

  • Aplicaciones web SPA (Single Page Applications).
  • Paneles administrativos y dashboards.
  • Aplicaciones empresariales de alto volumen de datos.
  • Portales dinámicos con manejo de rutas y estados.
  • Aplicaciones progresivas (PWA).

¿Sobre qué lenguajes se apoya?

  • TypeScript (lenguaje principal)
  • HTML
  • CSS / SCSS
  • JavaScript (para interoperabilidad)

¿Qué prerrequisitos necesita?

  • Node.js (v18 o superior)
  • npm (gestor incluido en Node)
  • Angular CLI
  • Conocimientos básicos de HTML, CSS y JS
  • Editor de código (VS Code recomendado)

2. Instalación

¿Se precisa instalación?

Sí. Angular requiere instalar Node.js y la CLI oficial.

Instalar Node.js

Descargar desde la web oficial e instalar con valores por defecto.

Comprobar la instalación:


node -v
npm -v

Instalar Angular CLI


npm install -g @angular/cli

Verificar:


ng version

Versiones necesarias

  • Angular CLI 17+
  • Node.js 18+
  • npm 9+

Rutas para tener en cuenta

La CLI generará un proyecto con la siguiente estructura base:


/src  
  /app  
  /assets  
angular.json  
package.json  
tsconfig.json

Variables de entorno

No requiere variables obligatorias para funcionar, pero opcionalmente se pueden configurar en:


src/environments/environment.ts

¿Necesita base de datos?

Angular NO incluye base de datos.

Puede conectarse a cualquier API REST (Node, Laravel, Java, Firebase, etc.).

3. Primeros pasos — “Hola Mundo”

1. Crear un nuevo proyecto


ng new mi-proyecto

Seleccionar las siguientes opciones:

  • ¿Usar routing? → Sí
  • Estilo de hojas → CSS (o SCSS)

2. Ejecutar el servidor


cd mi-proyecto
ng serve -o

Se abrirá en: http://localhost:4200

3. Mostrar “Hola Mundo”

Editar el archivo src/app/app.html:


<h1>Hola Mundo desde Angular!</h1>

4. Utilización del framework

Crear una aplicación base

Crear un componente:


ng generate component usuarios

Se generan 4 archivos:

 
usuarios.scss
usuarios.spec.ts
usuarios.ts
usuarios.html

Agregar contenido al HTML:

 
<h2>Listado de usuarios</h2>
<ul>
  <li>Usuario 1</li>
  <li>Usuario 2</li>
</ul>

Mostrar el componente en app.html:

 
<h1>Aplicación Angular Base</h1>
<app-usuarios></app-usuarios>

Es importante que en el archivo app.ts se añada la importación.

Debería quedar de la siguiente manera:


import { Component, signal } from '@angular/core';
import { Usuarios } from './usuarios/usuarios';

@Component({
  selector: 'app-root',
  standalone: true,
  imports: [Usuarios],
  templateUrl: './app.html',
  styleUrl: './app.scss'
})
export class App {
  protected readonly title = signal('mi-proyecto');
}

Visualizar resultados

Ejecutar:


ng serve

Abrir el navegador en:


http://localhost:4200

5. Explicación del funcionamiento del framework

¿Actúa sobre páginas, ficheros o secciones?

Angular trabaja principalmente sobre:

  • Componentes (unidades visuales)
  • Módulos
  • Servicios (lógica de negocio)
  • Rutas
  • Plantillas HTML

Cada componente representa una “parte” de la aplicación.

¿Cómo se editan los ficheros?

Composición de un componente:

  • HTML → Vista
  • TS → Lógica
  • CSS → Estilos

Ejemplo. En usuarios.ts indicamos en la parte de export los usuarios y importamos NgFor :


import { Component } from '@angular/core';
import { NgFor } from '@angular/common';

@Component({
  selector: 'app-usuarios',
  standalone: true,
  imports: [NgFor],
  templateUrl: './usuarios.html',
  styleUrl: './usuarios.scss',
})
export class Usuarios {
  usuarios = ["Ana", "Pedro", "Lucía"];
}

Y en usuarios.html :

 
<ul>
  <li *ngFor="let u of usuarios">{{ u }}</li>
</ul>

Por último modificamos usuarios.scss con una apariencia sencilla para que sea visualmente más atractivo.

 
ul {
  list-style-type: none;
  padding: 0;
  margin: 0;
  max-width: 300px;
  border: 1px solid #ccc;
  border-radius: 8px;
  overflow: hidden;
  background-color: #f9f9f9;
}

li {
  padding: 10px 15px;
  border-bottom: 1px solid #eee; /* Separador suave */
  font-family: Arial, sans-serif;
  font-size: 16px;
  color: #333;
  transition: background-color 0.2s;
}

li:last-child {
  border-bottom: none;
}

¿Cómo se referencian desde otras aplicaciones?

Angular puede exportar módulos reutilizables con:


@NgModule({
    exports: [Usuarios]
})
export class UsuariosModule { }

O consumir APIs externas mediante servicios.

Ejemplo:


this.http.get('https://api.example.com/users')
